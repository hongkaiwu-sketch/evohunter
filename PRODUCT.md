# Product

## Register

product

## Users

EvoHunter is used by recruiters, hiring operators, and builders validating an automated recruiting workflow. They work at a desk, compare candidates against a role, inspect AI extraction quality, and need quick confidence before acting on results.

## Product Purpose

EvoHunter turns JD text, candidate text, and public sources into GEP JSON, ranks candidates, and evolves scoring weights from feedback. Success means a user can move from raw input to a defensible candidate ranking without switching tools or manually reshaping JSON.

## Brand Personality

Precise, clinical, and quietly capable. The interface should feel like a working instrument for recruiting intelligence, not a marketing page or a decorative dashboard.

## Anti-references

Avoid landing-page heroes, oversized decorative cards, generic SaaS gradients, playful illustrations, glassmorphism, and dark sci-fi control rooms. Avoid UI that hides the raw JSON contract, because this project is still protocol-led.

## Design Principles

1. Keep the workflow visible from input to ranking.
2. Make AI output inspectable before downstream scoring.
3. Favor dense, predictable controls over decorative presentation.
4. Surface failure states in place with clear recovery paths.
5. Use color for action, selection, and status only.

## Accessibility & Inclusion

Target WCAG AA contrast, visible labels for every field, keyboard reachable controls, visible focus states, reduced-motion-safe transitions, and text-first status messages that do not rely on color alone.
